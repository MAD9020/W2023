# Week 9 In-class Exercise: Investigate mobile platforms

# Details

# Instructions

This exercise will be done in groups of 2.

Each group will investigate the main mobile phone platforms, by referencing their application design guidelines (shown below) and any other resources you can access, to find the following requirements for mobile phones for all listed PLATFORMS. For the purpose of this exercise, focus on the PHONES, not the tablets.

  • iOS: developer.apple.com/design/human-interface-guidelines/ios/

  • Android: m3.material.io

  • Fluent: developer.microsoft.com/en-us/fluentui#/

  • Polaris: polaris.shopify.com/

Use the attached template file to capture your findings. A few guidelines are done as an example of what's expected for the rest. Template: mobile_platforms_template_2023

  • Guidelines to be investigated: (Just follow the document attached above)
    1. key themes/principles of the platform
    2. default font
    3. text size (minimum, and recommended)
    4. application icon size
    5. in-app icon size
    6. branding recommendations
    7. notifications
    8. status bar
    9. transitions
    10. gestures
    11. alerts
    12. one 'unique' guideline per platform

# Deliverable for this exercise:

  • Follow the template provided to complete your work for submission
  • One submission per group: Include member names in the document
  • Filename: investigate_mobile_platforms.docx

Note

  • Some guidelines will be hard to find, and hard to understand.
  • Some guidelines don't exist for certain platforms.
  • If you can't find the exact piece of information, share the closest thing you were able to find, and indicate that you think it's not exactly right.
